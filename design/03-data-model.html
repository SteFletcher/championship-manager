The model is deliberately flat: there are no relational joins or ORMs. A Game instance owns arrays of Club objects, each owning an array of Player objects. Everything serialises to JSON by walking these arrays.

Derived finance

clubFinances(tier) sets starting balance = (tier/10)^3.4 × 3500 and transferBudget = 55% of balance, both rounded to the nearest £10k / £5k. So a tier-88 giant starts with millions; a tier-38 minnow with tens of thousands.

4. Player

The richest entity. Created by createPlayer(); developed yearly by developPlayer().

FieldTypeNotes
idstring<SHORT>-<index> or <SHORT>-Y<n> for youth.

listedbooleanTransfer-listed; cheaper to buy, more AI interest.
Ability & valuation

ability(player) is the headline number: pure def for GKs, 0.8/0.2 def/atk for DF, 0.8/0.2 atk/def for FW, and the mean for MF. It drives wages, values, AI interest, and XI selection. ageFactor ramps 0.75 → 1.0 across the peak window (24–28) then fades 0.13/year, so a 33-year-old star is worth far less than a 26-year-old of identical ability.

5. Match Entities

5.1 StatLine (per-match, per-player)

Lives inside MatchSim, keyed by player id. Never written back to the Player object during the match.

{
  id, name, pos, slot, started,
  minutes, rating,            // 1–10, base 6.0
  passes, tackles, shots, onTarget,
  goals, assists, saves, fouls,
  yellow, red, condition
}

5.2 MatchResult

Returned by sim.finish() and consumed by Game.advanceWeek().

{
  seed, homeTeam, awayTeam,
  score: {home, away},
  winner: 'home'|'away'|'draw',
  shootout: {home, away} | null,
  playedMinutes,
  events: [{minute, type, side, player, text}],
  stats: {home: SideStats, away: SideStats},
  scorers: {home: [{player,minute}], away: [...]},
  playerStats: {home: StatLine[], away: StatLine[]},
  injuries: [{side, id, player, minute, weeks}],
  subsUsed: {home, away},
  timeline: Snapshot[] | null
}

9. Validation Rules

validateTeam() enforces the structural contract for any XI handed to the match engine:

  • Exactly 11 players.
  • Exactly one GK.
  • Every position is one of GK/DF/MF/FW.
  • atk and def are finite and within [1, 99].
  • No duplicate player names within the team.

These guards throw descriptive errors before any simulation begins, which the test suite exercises explicitly.
