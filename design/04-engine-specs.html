04 · Engine Specifications

Detailed specifications for the simulation & game-loop subsystems

1. RNG Subsystem

Source: src/engine/rng.js. A mulberry32 PRNG with a serializable 32-bit state.

APIBehaviour
createRng(seed)Returns an RNG with state seeded from a u32.
next()Float in [0, 1).
int(min, max)Inclusive integer pick.
chance(p)Boolean, true with probability p.
pick(items)Uniform array pick.
weightedPick(entries)Picks {item, weight} by weight.
getState()/setState(s)Serialize/restore — used by save games.
hashString(str)FNV-1a 32-bit hash; derives seeds from club names.
Determinism proof

The test "identical seeds replay the identical match" deep-equals two simulateMatch results with the same seed. hashString guarantees the same club always produces the same squad, so a fresh career is reproducible from the manager name + club + seed alone.

2. Match Simulation Engine

Source: src/engine/match.js (867 LOC). The most complex subsystem.

2.1 Public API

class MatchSim {
  constructor(home, away, options)   // {seed?, knockout?, timeline?, homeAdvantage?, autoSubs?}
  playMinute()                       // → Event[] for that minute
  setTactics(sideKey, {formation?, mentality?})
  makeSub(sideKey, offId, onId)      // → {ok} | {ok:false, error}
  simulateToEnd()
  finish()                           // → MatchResult (see data model)
}
simulateMatch(home, away, options)   // convenience: one-shot → MatchResult

2.2 Setup Normalisation

The engine accepts three input shapes, normalised by normalizeSetup():

  1. Full squad (>11 players): selectXI auto-picks the XI + bench.
  2. Explicit XI ({name, players:[11]}): used as-is, slot = natural pos.
  3. Prepared lineup ({starters:[{player,slot}], bench, formation, mentality}): the richest form, used by the live UI.

validateSetup() then enforces exactly one GK slot, valid positions, and attribute ranges — throwing descriptive errors before any minute is played.

2.3 The Per-Minute Loop

playMinute():
  minute++
  if minute == 46: log half-time
  for each side: #aiTacticsPolicy ; #autoSubPolicy
  resolve possession (midfield battle × home boost)
  quiet build-up: passes (3–8) + maybe a tackle  → feeds live ratings
  fouls & cards (per side, 11% chance):
      straight red 0.6% · yellow 11% · second-yellow → red
  injuries (0.18% per side/min, weighted by injuryProne & low condition):
      AI auto-subs immediately; user side left short until manager reacts
  chance resolution (BASE_CHANCE_PROB × mentality mods × atk/def ratio):
      shooter picked by SHOOT_WEIGHT (FW=3, MF=1, DF=0.3)
      miss (42%−atk×0.15%) · block (22%) · on-target → goal vs save
      assist awarded 60% (ASSIST_WEIGHT MF=2, FW=1.5, DF=0.5)
  condition decay (GK 0.08/min, others 0.30/min)
  push timeline snapshot
  if minute ≥ finalMinute: #endStage (ET? shootout? complete)

2.4 Strength Model

Unit strength is averaged over the players occupying each slot, using effective contribution:

#eff(side, entry, attr) = player[attr]
                       × slotPenalty(player, slot)   // 1.0 natural, 0.75 OOP, 0.5 GK mismatch
                       × sharpness                   // morale + form + consistency noise
                       × (0.7 + 0.3 × condition/100)

Headcount matters with diminishing returns: (slotCount / baseline)^exponent, where midfield uses exponent 0.25 (gentle possession shift) and defence/attack use 0.5. This is why 5-3-2 defends and 4-3-3 attacks without midfield swamping possession.

2.6 AI Behaviour

  • Tactics policy (after minute 55, 6% chance/min): trailing → attacking (and 4-3-3 after 75); leading after 70 → defensive (5-3-2 after 82); level & short-handed → defensive.
  • Auto-sub policy (60–85 min, 5% chance): replaces the most tired outfielder (<80 condition) with the best bench fit for the slot.
  • Forced sub on injury: AI sides substitute immediately; user sides are left a man short to force a managerial reaction (the auto-pause).
  • Emergency keeper: if a keeper is lost with no GK on the bench, the best-defending outfielder pulls on the gloves.

2.7 Knockout Resolution

When knockout: true and level at 90+stoppage, 30 minutes of extra time (plus 1–2 min stoppage) are appended. If still level, a best-of-5 penalty shootout runs, finishing early once decided; sudden death follows if needed. Penalty conversion: clamp(0.76 + (taker.atk − oppKeeper.def) × 0.0015, 0.55, 0.92).

3. League & Cup Subsystem

3.1 Fixtures (league.js)

The circle method generates a double round-robin. For n clubs it produces 2(n−1) rounds of n/2 matches, with the fixed team's venue alternating to balance home/away. A null "bye" marker handles odd counts.

3.2 Table (computeTable)

Accumulates played/won/drawn/lost/goalsFor/goalsAgainst/goalDiff/points, then sorts on points → goalDiff → goalsFor → name. Three points for a win, one for a draw.

3.3 Cup (cup.js)

Single-leg knockout spanning all 40 clubs. The opening round reduces the field to the largest power of two below the entrant count (40 → 32) via byes drawn with Fisher-Yates using the shared RNG. Ties then halve each round to the final. Round names map by survivor count (Final, Semi, Quarter, Round of 16, First Round). advanceCup() records winners and redraws; inCup() tests liveness.

Power-of-two reduction

openingTarget(n) = 2^floor(log2(n−1)). So 12→8, 16→8, 24→16, 40→32. This keeps every subsequent round a clean halving, avoiding awkward byes deep in the bracket.

3.4 Calendar (season.js)

buildCalendar(leagueRounds, cupRounds) emits one league entry per round, inserting a cup day after the round closest to rounds × (i+1)/(cupRounds+1) for each cup round — spreading the cup evenly through the season. Months are 4 weeks; monthName(week) drives Player of the Month.

4. Transfer Subsystem

Source: src/engine/transfers.js (pure functions); glued to state in game.js.

4.1 Valuation

FunctionFormula / rule
askingPrice(club, player)value × (listed?0.85:1.2) × (key player?1.25:1), rounded to £5k. Key player = within 3 of squad best ability.
evaluateBid(club, player, amount)≥ asking → accepted; ≥ 90% value → countered at midpoint; else rejected.
wageDemand(player, fromTier, toTier)max(wage×1.1, fairWage); ×1.35 to step down a tier.
freeAgentWage(player)fairWage × 1.05.

4.2 Player Willingness

playerAgrees(rng, player, fromTier, toTier): always agrees if toTier ≥ fromTier − 4 (lateral or upward). Otherwise reluctant, more so at peak age, capped at 85% rejection. This stops stars dropping to minnows without friction.

4.3 Squad Protection

canRelease(club, player) refuses if the squad would drop to <15 players, or if selling a GK would leave fewer than 2. This prevents the AI or the user from gutting a squad below viability.

4.4 AI Market Behaviour

  • Interest: aiInterest(club, player) = upgrade over the club's best in that position (+ thin-squad bonus). Weighted pick selects the target.
  • Offers: aiOfferAmount = value × (0.9..1.4).
  • Activity: ~22% weekly chance of an AI bid for a user player; ~15% weekly chance of an AI-to-AI transfer. Listed players attract 2.5× interest.

4.5 Scouting & Attribute Masking

Game.attrDisplay(player, attr) returns exact attributes for the user's own players and scouted players; for everyone else it returns a stable range [lo, hi] of half-width 8 that contains the truth but is offset by a per-player deterministic hash so the centre never leaks the real value. Sorting uses visibleMid() so the market orders by what the manager can see. A £15k scout report (1 week) flips a player to "known exactly".

5. Career Loop (game.js)

5.1 advanceWeek(userResult)

The master tick. Steps:

  1. Resolve the calendar event: league round (both divisions) or cup round.
  2. For each pairing, #resultFor uses the user's live result if it's the user's match, else simulateMatch with a fresh seed from rng.
  3. #applyMatchResult: morale/form/condition/injuries/bans, fanbase drift, form guide, season stats, gate receipts & attendance.
  4. #weeklyUpkeep: wages deducted, injuries ticked, condition recovered +18.
  5. #tickExpansions & #tickScouts.
  6. #aiTransferActivity & #maybeAiOfferForUserPlayer.
  7. #updateBoard: confidence from league drift + result; sack if < 15.
  8. week++; monthly awards every 4 weeks; season end if calendar exhausted.

5.4 Board & Job Market

  • Confidence: drift = clamp(expectation − position, −2, +2); result delta +3/0/−4. Clamped [0, 100]. < 15 → sack.
  • Sack: reputation −12; up to 2 job offers from clubs with tier ≤ reputation + 15.
  • Poach: after a strong season, 40% chance a club with tier > club.tier + 5 and ≤ reputation + 15 comes calling.
  • Reputation deltas: champion +12, cup win +8, promotion +8, relegation −8, sack −12.

5.5 Season End

A large deterministic transition. In order: final tables; prize money; fanbase pull; promotion/relegation; season awards (golden boot, player of season); board verdict & reputation; sack-or-poach check; player development & retirements; contract expiries (user must renew deliberately; AI retains 70%); youth intake to ≥16; AI expansions; expectation re-ranking by squad strength; calendar & cup reset; seasonIndex++.
