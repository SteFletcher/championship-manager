Strategy, coverage matrix, and the realism harness

1. Test Strategy

Testing follows the architecture's purity boundary: because the engine has no DOM dependency, the bulk of correctness is proven with fast, deterministic node --test unit and property tests. A single puppeteer-driven browser smoke test then proves the UI wires the engine correctly end-to-end.

4. Statistical Realism Harness

The match engine is a stochastic system, so a single match proves little. The suite instead asserts distributional properties over thousands of simulations:

What the harness checks
  • Score ≡ goal events. Across 500 matches, the final score always equals the count of goal-typed events, side by side.
  • Shot funnel. Shots ≥ shots on target ≥ goals, for every side in every match.
  • Possession sums to 100. Home + away possession always totals 100.
  • Sent-off players never reappear. Once removed, a dismissed player is absent from the pitch for all subsequent minutes.
  • Goal rates. Over thousands of sims, average goals per match sits in a believable band (~1.3–2.4), with home teams scoring more than away teams.
  • Mismatch dominance. A 90-rated side beats a 55-rated side the large majority of the time.
  • Determinism. 50 different seeds produce ≥45 distinct matches (no degenerate repetition), while the same seed always produces identical output.

5. The Browser Smoke Test

test/browser-smoke.js is intentionally not part of npm test — it requires a local Chromium-family browser and the dev server running on :8123. It:

  1. Launches a headless Brave/Chromium via puppeteer-core.
  2. Navigates to the app and starts a career at a known club.
  3. Drives the Continue → pre-match → kick-off → full-time flow.
  4. Advances several weeks and asserts DOM invariants (e.g. the HUD updates, the table has the right number of rows).
  5. Optionally dumps screenshots when SHOT_DIR is set — which is exactly how the screenshots in 05 · UI Design were produced.
# Run the E2E smoke (server must be running)
npm run serve            # in another terminal
node test/browser-smoke.js [path-to-browser-binary]

6. Save/Restore Determinism Test

A particularly important integration test, given the determinism contract:

  1. Play N weeks of a career from a seed, recording the result after each week.
  2. Serialize the game, restore it, and continue playing.
  3. Assert the resumed career produces exactly the same sequence of results as an uninterrupted run from the same seed.

This catches any state that fails to round-trip through JSON (e.g. Maps, undefined fields, class brands) and any RNG consumption that isn't captured by rngState.
