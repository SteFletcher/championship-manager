12 · Match Tactics Re-Design

Drag players between positions — on the tactics screen and live during a match — and make the paused match a proper tactics view: subs, formation, mentality, instructions, and positioning in one place

✅ Shipped 2026-07-16

Built as designed, same day: pointer-drag swaps on both pitches, MatchSim.swapPositions with five dedicated tests (218 total), the paused-match tactics board, and drag flows exercised by real mouse events in the browser smoke. The bench-drag substitution remains the named follow-up.

1. Problem

EE-2 gave formations real geometry and EE-4 made players clickable, but positioning is still indirect: the only way to put a specific player in a specific slot is the click-two-rows swap on the tactics tables, and mid-match there is no way at all — a formation change auto-remaps players and the manager cannot correct it. The pause panel groups subs and dropdowns away from the pitch, so the pitch reads as a display, not a control surface.

3. Interaction Design

Pointer state machine (per own-side marker, active whenever the pitch is editable —
always on the tactics screen; only while paused on match day):

  pointerdown ──► armed (record x,y; capture pointer)
  armed ── move < 6px ──► pointerup ──► CLICK  → instruction panel (EE-4)
  armed ── move ≥ 6px ──► DRAGGING (marker follows pointer, z-top, ghost left in slot)
  DRAGGING ── over own outfield marker ──► target highlights (gold ring)
  DRAGGING ── pointerup on target ──► SWAP (players exchange slots; instructions stay)
  DRAGGING ── pointerup elsewhere / Esc ──► revert, no change
  • Swap, not free placement. Slots are the formation's geometry; players move between slots. Dropping on empty grass does nothing — this keeps the XI always valid (11 slots, one keeper) with no new failure modes.
  • Instructions stay with the slot. Dragging your winger to right-back leaves the "forward runs" instruction on the right-back slot — the same rule substitutions and formation remaps already follow, so there is exactly one mental model.
  • Familiarity feedback. After a swap, the existing amber out-of-position tags and the familiarity multiplier apply unchanged; the marker label (slot code) is the truth of where each man now plays.

4. Engine Design

// match.js
swapPositions(sideKey, idA, idB)
  → { ok } | { ok: false, error }
  · both players on the pitch, distinct; neither slot is GK
  · entries exchange slot AND instr/mods (instructions belong to the slot)
  · stat-line .slot fields updated; slotCounts unchanged (same slot set)
  · logs type 'position', data { aId, bId, aSlot, bSlot } (post-swap slots)
  · zero RNG draws — replaying a scripted swap is byte-identical

Pre-match, no engine change is needed: the tactics screen already owns the lineup, so a drag is the existing starters-swap expressed on the pitch instead of the tables.

6. Out of Scope / Cut Line

Not buildingWhy
Free-form marker placementSlot geometry is the formation; free placement belongs to a future formation editor.
Drag bench player onto the pitch to substituteNice-to-have; the sub lists already work. Named follow-up if the drag plumbing proves itself.
Touch-screen long-press affordancesPointer events cover touch basically; dedicated mobile polish is a separate pass.

Cut line: DRG-01/03/04/05 (tactics-screen drag + engine API) ship even if the match-day drag slips; the pause-panel retitle is trivial and ships regardless.

7. Acceptance

Done when
  1. A player can be dragged between positions on the tactics screen and, while paused, on the live match pitch — with the swap visible in slot labels, familiarity tags, and (in-match) the commentary.
  2. Change formation mid-match, then drag players into the exact slots you want — instructions behave per the EE-4 slot rules throughout.
  3. Click-to-instruct still works exactly as before.
  4. The full suite and browser smoke are green; a scripted career replays identically.
