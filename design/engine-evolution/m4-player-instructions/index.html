EE-4 · Per-Player Instructions

Click a player on the tactics pitch and tell him what to do — forward runs and pressing, persisted with tactics, drawn as arrows, and honoured by the engine

✅ Shipped 2026-07-16 · save v9

Built as designed (commit b9f26d7): both instruction axes per formation slot, engine modifier plumbing proven byte-identical at defaults before the AI presets landed (goldens then re-pinned once, with reason), the tactics-pitch click panel with arrows and balance strip, the same panel at any match pause, AI presets on in-match mentality switches, and a deterministic v8→v9 migration. Suite at 213 tests. One deviation from the spec text: secondary-position familiarity is 0.9 (per EE-2 as built), and the mid-match API keys off the player currently wearing the slot (setInstruction(side, playerId, axis, value)).

1. Purpose

This is the interaction that defined FM's takeover of CM: tactics stop being a team-wide dial and become per player. Click the right-back → "get forward". Click the ageing striker → "sit and hold". The engine consumes instructions as modifiers on contribution weights, so an attacking full-back genuinely creates more and defends less. Two instruction axes keep the scope honest; the plumbing is built so later axes (marking, passing style) are additive entries in one table.

2. Session Scope

In scopeOut of scope (deferred)
Two instruction axes (runs, pressing) per formation slot; engine modifier plumbing; tactics-pitch click panel + arrows; persistence with saved tactics; mid-match changes at any pause; AI presets by mentality; save migration. More axes (marking, passing style, width); per-opponent saved tactic sets; drag-repositioning of slots; instruction effects on set pieces; instruction-driven commentary lines (EE-6 does commentary properly).

Cut line: engine modifiers + squad-list instruction editing ship even if the pitch click-panel slips; the panel is the headline UX though, so it is sequenced first in the build order, not last.

4. Engine Design

4.1 One modifier table, consulted at existing decision points

// team.js
export const INSTRUCTIONS = {
  runs:  { forward: { atkC:1.15, defC:0.85, shootW:1.5, assistW:1.4, decay:1.15 },
           balanced:{},                                  // all 1.0
           hold:    { atkC:0.87, defC:1.12, shootW:0.6, assistW:0.7, decay:0.9 } },
  press: { high:   { tackleW:1.4, foulCh:1.2, oppPass:-4, decay:1.10 },
           normal: {},
           deep:   { tackleW:0.7, foulCh:0.85, oppPass:+3, concedeQ:0.95 } },
};
instructionMods(slotInstr) → merged {atkC, defC, shootW, assistW, tackleW, foulCh, oppPass, decay, concedeQ}

Integration points, all existing: #eff multiplies atkC/defC into the attribute it is computing (attack-side vs defence-side split per unit-strength kind); #pick weights multiply shootW/assistW/tackleW; the build-up pass roll adds the defending side's summed oppPass (scaled by pressers on pitch); fatigue phase multiplies decay. Because defaults are exact 1.0/0-point no-ops, a default-instruction match reproduces today's engine — that is the regression stance (PTI-08).

4.2 API & persistence

// game.js
setSlotInstruction(slotIndex, axis, value)   // validates, persists in game.tactics
// game.tactics gains: instructions: [{runs, press} × 11] aligned to formation slots
// MatchSim setup gains: starters[i].instr — normalizeSetup defaults balanced/normal
// MatchSim.setInstruction(sideKey, slotIndex, axis, value) — mid-match, logs a
// 'tactics' event, no RNG draw

Formation changes remap instructions with the slot (keeper excluded); a formation with fewer compatible slots resets orphaned entries to defaults and says so in the tactics log line.

8. Exit Criteria

EE-4 is done when
  1. A player can be clicked on the tactics pitch, given instructions, and the pitch shows the arrows — pre-match and at any in-match pause.
  2. The statistical tests prove instructions change play in the promised directions and magnitudes.
  3. Default-instruction matches replay byte-identical to pre-milestone goldens.
  4. Old saves load with balanced defaults and play on.

9. Risks

RiskMitigation
