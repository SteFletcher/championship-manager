CHAMPIONSHIP MANAGER

A retro football-management career game that runs entirely in the browser, modelled on Championship Manager 99/00. This package documents the application as if it had been specified up-front as a waterfall project — requirements, architecture, data model, engine specifications, UI design, test plan, delivery schedule, risk register, and a forward roadmap.

Product at a Glance

Elevator pitch

Take charge of one of the 40 real 1995/96 English clubs — genuine period squads across the Premier League and Division One — and manage squads, tactics, transfers, finances and the board across full seasons of league and cup football — all in the browser, with a resumable minute-by-minute match engine you can pause to make substitutions and tactical changes. No build step, no backend; games auto-save to localStorage and replay deterministically from their seed.

How to read this package

The documents are ordered as a waterfall delivery: requirements flow down into architecture and data model, which flow into engine and UI specifications, which flow into the test plan and project schedule. Each document is self-contained but cross-references the others. The UI Design document contains real screenshots captured from the running application; the Roadmap looks forward from the current feature set, and Horizon 1 Specs details the designs for its near-term items.

Reverse-engineering note. This package was produced by reading the existing codebase and lifting its design into the form a waterfall project would have produced up-front. Where the code reveals a deliberate decision, that decision is documented as a requirement or design choice. The roadmap is the only forward-looking section; everything else describes what is.

Status update (2026-07-15). Since the package was written, the world data moved to the real 1995/96 English leagues and squads, and the first three Engine Evolution milestones shipped: EE-1 (phase-pipeline core + golden-master/statistical regression armour), EE-2 (ten detailed positions, formation geometry, familiarity — save v7), and EE-3 (eight real attributes driving the engine — save v8). The facts above reflect the current build; the deep-dive docs 03/04 carry inline notes where the EE milestones changed the original design.
